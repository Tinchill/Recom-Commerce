# 面向电商精准营销的高潜用户购买预测系统

## 1. 项目概述

本项目是一个完整的、面向电商精准营销场景的端到端推荐系统。项目核心目标是**基于用户历史行为数据，精准预测用户在未来一段时间内对特定品牌的购买意向**，从而筛选出高潜力用户群体（如Top 10万），以支持短信、App Push等精准营销活动，提升营销转化率（ROI）。

整个项目基于**阿里巴巴天池T-mall数据集**，在**阿里云平台**上完成。技术上，实现了从数据仓库构建、特征工程、样本生成到深度学习模型（DNN, DIN, MOE）训练与评估的全流程。

## 2. 技术栈

*   **云平台与大数据:** 阿里云 (MaxCompute, DataWorks, PAI-DSW)
*   **数据处理与仓储:** SQL (MaxCompute SQL)
*   **模型开发与训练:** Python, PyTorch, Pandas, Numpy
*   **核心算法:** Base DNN, DIN (Deep Interest Network), MOE (Mixture-of-Experts)
*   **核心优化:** Focal Loss (解决样本不均衡问题)

## 3. 项目核心亮点

*   **端到端全流程实践:** 独立完成了从原始数据处理、数据仓库分层（ODS, DW, ADS）、特征体系构建、样本拼接，到多种深度学习模型的设计、实现、训练、评估和优化的完整机器学习工程链路。
*   **工业级数据架构:** 基于MaxCompute设计并实现了标准的三层数仓架构，展现了处理海量数据（千万级用户、亿级行为）的工程能力和规范化开发思想。
*   **前沿算法应用与优化:** 不仅实现了经典的DNN模型，更深入实践了工业界广泛应用的**DIN模型**（捕捉用户动态兴趣）和**MOE模型**（处理用户群体异构性），并针对购买预测中的**严重样本不均衡问题**，引入并实现了**Focal Loss**进行针对性优化。
*   **严谨的实验与评估:** 建立了完善的评估体系，通过对比不同模型与损失函数的组合在**AUC**和**Top-N召回率**上的表现，科学地验证了各模块的有效性，并最终得出了最优模型方案。

## 4. 系统工作流

整个项目分为两大核心流程：**数据处理流程（基于DataWorks & MaxCompute）** 和 **模型训练评估流程（基于PAI-DSW）**。

### 4.1 数据处理流程 (SQL)

此流程的代码位于 `feature_process/` 目录下，按数字顺序执行，清晰地构建了从原始数据到最终模型输入的全过程。

1.  **数据仓库构建 (`1_data/`)**:
    *   `ods_data/`: 原始数据加载层，将数据集导入MaxCompute表。
    *   `dw_data/`: 数据仓库明细层，按行为类型（点击/收藏/加购/购买）和天分区存储用户行为日志 (`user_beh_log_dw.sql`)。
    *   `dim/`: 维度表层，构建商品、品牌、类目等维表 (`brand_cate_dim.sql`, `item_dim.sql`)。
    *   `ads_data/`: 数据应用服务层，聚合计算三类核心特征：用户侧特征、品牌侧特征、用户-品牌交叉特征 (`user_beh_feature_ads.sql`, `brand_stat_feature_ads.sql`, etc.)。

2.  **特征与样本生成 (`2_feature_and_sample/`)**:
    *   定义正负样本：以未来7天内是否购买某品牌为标签 (`user_pay_sample.sql`)。
    *   构建序列特征：为DIN等模型生成用户行为序列特征 (`user_brand_seq_feature.sql`)。
    *   特征与样本拼接：将生成的特征与样本关联，形成模型训练的宽表 (`user_pay_sample_feature_join_dnn.sql`)。

3.  **训练/评估集准备 (`3_train/`, `4_eval/`)**:
    *   划分并随机打乱训练集 (`user_pay_sample_feature_seq_shuffle.sql`)。
    *   生成固定的评估集 (`user_pay_sample_feature_join_dnn_eval.sql`)。

### 4.2 模型训练评估流程 (Python)

此流程的代码位于 `model_train_eval/` 目录下，结构清晰，各模块解耦。

*   **配置 (`config/`)**: 存储阿里云AK配置及模型超参数 (`ak_config.py`, `dnn_config.py`)。
*   **数据加载 (`dataset/`)**: 负责从MaxCompute读取处理好的数据，并封装成DataLoader (`dnn_dataset.py`)。
*   **损失函数 (`loss/`)**: 存放自定义的损失函数，如 `focal_loss.py`。
*   **模型定义 (`model/`)**: 存放所有深度学习模型的网络结构定义 (`dnn_model.py`, `din_model.py`, `moe_model.py`)。
*   **训练与评估脚本**:
    *   `dnn_model_train.py`, `din_model_train.py` 等脚本是模型训练的入口。
    *   `dnn_model_test.py` 是模型在测试集上进行评估的入口。
    *   通过组合不同的训练脚本（如 `moe_focal_loss_model_train.py`），可以方便地进行模型和损失函数的组合实验。

## 5. 模型性能与成果

通过一系列对比实验，我们验证了不同模型结构和损失函数对最终效果的影响，核心指标为AUC（衡量整体排序能力）和Recall@5000（衡量头部高潜力用户的发现能力）。

| 模型       | 损失函数       | AUC (参考值) | Recall@5000 (参考值) | 分析                                            |
| :--------- | :------------- | :----------- | :--------------------- | :---------------------------------------------- |
| Base DNN   | Cross Entropy  | `0.82`       | `15%`                  | 基准模型，性能基础。                            |
| Base DNN   | **Focal Loss** | `0.82`       | `18%`                  | **召回率提升**，有效缓解样本不均衡问题。        |
| **DIN**    | Cross Entropy  | `0.85`       | `20%`                  | **AUC显著提升**，Attention机制精准捕捉用户兴趣。 |
| **DIN**    | **Focal Loss** | `0.85`       | `24%`                  | **强强联合**，实现AUC和召回率的最佳平衡。      |
| **MOE**    | Focal Loss     | `0.86`       | `25%`                  | **极致召回**，验证了处理用户群体异构性的潜力。 |

**最终成果:**

最终采用的 **DIN + Focal Loss** 模型方案，相比基线DNN模型，在**AUC上提升了0.03**，在关键的**Top-N召回率上实现了约60%的相对提升** (`(24%-15%)/15%`)。这证明了模型在精准发现高潜力用户方面的巨大价值，能有效支撑后续的精准营销投放，提升业务ROI。
